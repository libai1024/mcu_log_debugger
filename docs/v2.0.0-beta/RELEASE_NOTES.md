# v2.0.0-beta

## Highlights
- 全面升级 UI/UX，布局与交互更符合现代桌面工具的使用习惯。
- 增强日志过滤与搜索能力，为后续 v2.x 功能打下基础。
- 引入企业级 CSS 拆分方案，显著提升样式可维护性。

---

## New Features

### 日志过滤与交互
- **日志级别多选下拉**
  - 支持展开/收起、全选/清空操作。
  - 勾选后实时刷新日志表格。
  - 触发器文本使用彩色 Chips（V/D/I/W/E）展示当前选中级别摘要。
- **Tag 过滤增强**
  - 使用事件委托，点击 Tag 文本、按钮或周围空白区域都能切换选中状态。
  - 提供“全选/取消全选”按钮，快速切换过滤范围。

### 保存与导出闭环
- **导出按钮打通后端**
  - 点击导出会通过 Tauri 命令调用后端，将当前过滤后的日志内容写入文件。
  - 支持多种导出格式（TXT/CSV/JSON），默认格式可在设置中配置。
- **设置中的保存路径**
  - 补齐“保存路径”字段读取/写入逻辑。
  - 支持通过系统对话框选择目录，并保存到应用状态与本地存储。

### 搜索体验
- 侧栏搜索区结构优化：
  - 第一行：输入框 + “上一个/下一个”跳转按钮。
  - 第二行：搜索模式按钮（区分大小写、全词匹配、正则）+ 结果计数 `0/0`。
- 支持以下搜索模式：
  - 区分大小写 (Case Sensitive)
  - 全词匹配 (Whole Word)
  - 正则表达式 (Regex)

## Improvements

### 样式与架构
- 将原来的 `public/style.css` 拆分为多层：
  - `public/styles/theme.css`：主题和设计 token（颜色/字体/间距等）。
  - `public/styles/base.css`：基础样式与 Reset。
  - `public/styles/layout/**`：App 布局、标题栏、侧边栏等布局相关样式。
  - `public/styles/components/**`：按钮、下拉框、多选框、搜索框等通用组件样式。
  - `public/styles/features/**`：日志表格、统计条、发送面板、弹窗、通知等业务特性样式。
  - `public/styles/core/animations.css`：全局动画关键帧集中管理。
- 使用统一的激活态 token（`--primary-active-*`）为搜索模式按钮等可切换控件提供统一高亮风格。

### 细节优化
- 日志级别多选下拉：选中状态高亮选项行背景，提升可读性。
- Tag 列表 hover 与 active 状态统一，视觉反馈更明确。
- 收藏夹角标尺寸与定位优化，为后续 UI 整体风格统一打基础（v2.0.2 中继续完善）。

---

## Breaking Changes
- 样式文件路径发生较大变化：
  - 不再建议直接引用 `public/style.css`，应通过 `index.html` 中的分模块 CSS 引用加载样式。
- 如有脚本或第三方工具依赖旧样式路径，需要同步更新到新的结构。

---

## Known Issues
- 在某些环境中（首次加载、系统字体/图标尚未完全就绪），模式指示条位置可能会出现极短暂的抖动，但不会影响功能使用。
- Windows/Linux 下的打包与安装流程仍在优化中，本版本主要面向 macOS 开发环境验证。

---

## Upgrade Notes
- 建议在从 v1.x 升级到 v2.0.0-beta 前，备份旧版本配置与日志目录。
- 升级后，如遇样式异常，可尝试清空缓存后重新启动应用。
